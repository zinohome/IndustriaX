# IndustriaX 平台 PRD v1

> 本文档由确定的技术选型反推得出——不是先有需求再选型,而是从已锁定的组合反向还原出它隐含的产品意图与约束。用于选型阶段收口、进入架构设计前的对齐基线。

---

## 0. 一句话定义

一个**小模型驱动、可私有化部署、松散可组合**的工业 AI 底座:把杂乱的工业文档与流程,转化为可靠、可追溯、可审计的 agent 工作流;其中沉淀的核心资产(工业解析、合规知识图、Skill 库)跨客户复用,整套运行在客户防火墙内、数据不出厂。

---

## 1. 为什么要做(反推出的问题)

工业企业要落地 AI,卡在三件事上,而这三件正好解释了为什么选型长成现在这样:

1. **数据不能出厂。** 工业客户(尤其涉及变更、合规、工艺)的数据敏感,云端大模型 API 进不去。→ 这逼出了"本地小模型 + 私有化"的全部技术约束。
2. **文档是脏的、流程是长的。** 工程图纸、FA 报告、ECN/PCN、BOM、合规证书,标准 RAG 解析不动;变更流程跨天跨周、要审计、不能丢状态。→ 这逼出了 RAGFlow(解析)+ Temporal(持久编排)。
3. **每个客户都重做一遍不可持续。** 项目制交付没有复利。→ 这逼出了"反项目制"原则:可复用的部分(解析能力、合规知识、Skill)做厚,不可复用的水管全借。

**结论:这不是"做一个 AI 应用",是"建一个能跨客户复用、且能进客户内网的工业 AI 生产底座"。**

---

## 2. 目标与非目标

### 目标
- 在客户**完全私有化**(数据不出厂、可断网)的前提下跑通工业 agent 工作流。
- 主体推理用**本地小模型**(Qwen3 7B/14B 级),靠结构而非模型规模拿到工业级可靠性。
- 沉淀**三块可跨客户复用的资产**:工业文档解析硬化、合规/变更知识图、工业 Skill 库;目标复用率 60–75%。
- 每个客户的差异只体现在**组合方式**,不体现在重写。
- 整套 License 干净(Apache/MIT),可进企业尽调、可白标交付。

### 非目标
- 不做面向非技术用户的拖拽式低代码平台(那是 Coze/Dify 的活)。
- 不做通用聊天机器人 / 拟人陪伴(记忆层刻意做"单纯",不做人格化)。
- 不自建模型、不自建工作流可视化编排器、不自建任何已商品化的基础设施水管。
- 不追求"纯小模型 100% 全包"——保留升级口给硬尾巴。

---

## 3. 目标用户与切入场景

**直接用户:** 工业企业内部的工艺、研发、质量(FA)、供应链、IT 等角色,以及对接它们的 BD/交付团队。

**切入楔子:** ECN/PCN 变更管理——流程标准、痛点明确、数据有 schema、可追溯需求硬。

**长期终局:** 三边统一决策图——选型/报价 ↔ 变更管理 ↔ 市场准入/合规;其中**合规/认证是最强的跨客户知识护城河**(因为法规公开,知识可沉淀、可复用)。

---

## 4. 核心设计原则(整套选型的共同前提)

1. **智能压进结构,不押注模型。** 凡能搬下推理时模型的(解析、多跳、编排),全搬到专用模型 / 图遍历 / 确定性代码 / 离线批处理。推理时的小模型只留最薄一层。
2. **松散可组合。** 每个能力是独立服务、走 MCP 通信、可单独替换、可按客户重组。任何一块能换。
3. **知识与记忆分离。** 权威知识(合规、规格、证书=ground truth)进 RAG/KB 层且**人工策展**;agent 学到的低可信信息进 Memory 层。两者信任级别不可混。
4. **借水管,建资产。** 会随客户复用的自己建(解析硬化、合规图、Skill);不会复用的全借。
5. **宁可弃权,不要猜。** 工业场景一个自信的错答案 > 没有答案。抽取式 + 可追溯 + "查不到就转人工"。

## 6. 分层功能需求

### 6.1 Memory(Mem0)
- 存"这个 agent 本次任务做了什么、学到什么、用户偏好",按 user/agent/session 隔离。
- 写入用规则或小模型轻量抽取;**整合/去重/冲突消解放离线批处理**(可用大模型跑)。
- 读取纯检索,零 LLM。
- **硬约束:不承载权威知识,不做人格化,不做时序推理(那属于知识图)。**

### 6.2 RAG(RAGFlow + Postgres/AGE)
- **解析层(RAGFlow / DeepDoc):** 工业 PDF、扫描件、表格、图纸、DOCX 的版面感知解析;每个答案带来源段落(合规硬需求)。这是要花力气硬化的地方。
- **结构化:** ECN/PCN、BOM、FA 报告按 schema 抽成结构化数据,而非纯切块。
- **图层(Postgres + Apache AGE):** 零件↔变更↔证书↔规格的**人工策展**属性图;多跳由图遍历**预先算好**,小模型只读结果。
  - 仅放策展的合规/变更图,不堆杂关系。
  - 索引策略须显式设计(AGE 默认不建索引,是唯一会真出问题的点)。
  - 升级触发条件:图达百万节点级 / 频繁 5+ 跳深遍历 / 需水平分片——任一成立才考虑拆出去上 Neo4j(走网络协议隔离 GPL)。
- **图的边只允许人工策展或可验证来源,禁止 LLM 自动抽边**(一条错边=一个错的监管答案)。

### 6.3 推理(Qwen3 本地 + router)
- 主体:Qwen3 7B/14B 本地(4090),做抽取、分类、填模板,**不做自由长篇合成**。
- 查询理解降级为意图分类 + 槽位填充,不做自由 query rewrite。
- 生成降级为"用检索结果填模板",抽取式 + 可追溯。
- router:命中硬 case(无预结构化的长链多跳、合规判断、长篇合成)升级。
- **升级口为按客户分级的可配置开关** `escalation_target`,由客户数据外发策略决定,非全局拍死:
  - `local_large`:硬密封客户——只升级到内网更大本地模型;再不行转人工。
  - `external_api`:可接受外发客户——升级到 OpenAI 兼容 API(成本远低于内网再部署大模型)。Pydantic AI 模型无关,切换仅配置变更,不动架构。
  - `human`:不升级,直接转人工。
- **数据边界过滤(external_api 模式必备):** 升级到外部 API 前,按数据级别过滤,规则见 §6.8。router 设计须预留过滤位,不做布尔一刀切。
- 全模型调成"宁可弃权,不要猜"。

### 6.4 编排(Temporal)
- 工业流程的执行引擎:长流程、持久化、崩溃恢复、全程审计。流程由工程师 code-first 定义。
- **不用 LLM 编排框架做执行大脑**(它们没持久化)。
- 界面:先用 Temporal 自带开源 UI 套权限给客户看(1–2 周);不够再自研业务语义监控台(把技术 workflow 状态翻译成"变更申请→评审→生效"的业务视图,1–2 个月)。
- **不做可视化拖拽编排器**(客户不需要自编排,且偏离主线)。

### 6.5 Harness(Pydantic AI)
- agent 薄壳:管 loop、类型安全 I/O、加载 Skill、调 MCP 工具。
- 原生驱动本地 Qwen3(经 Ollama/兼容接口),不缝 Claude 适配层。
- 类型安全 + 结构化输出,贴小模型 + 重约束路线。
- **待办:SKILL.md 是 Anthropic 约定,Pydantic AI 不原生支持,需在其 capability 机制上自建 SKILL.md 加载器**(设计阶段排工时)。

### 6.6 Skill(SKILL.md + MCP)
- 把每个能力(ECN/PCN 处理、合规查询、FA 分析、BOM 解析)封装成 SKILL.md 包 + MCP server,可热插、可按客户重组。
- 借格式(SKILL.md + MCP),**内容全自建**——这是 60–75% 复用的载体,是核心资产。

### 6.7 粘合(MCP)
- Memory、RAG、每个 Skill、外部工具都包成 MCP server;Harness 在上面组合,Temporal 在下面保执行。
- 协议统一,任何一块可替换。

### 6.8 数据分类与跨边界规则(external_api 模式)
> 不自创分类体系,直接对齐国家/行业法定框架(客户本身大概率已在执行),作为 router 数据边界过滤的依据。

**两层框架(套用工信部《工业数据分类分级指南》+《工信领域数据安全管理办法》+ GB/T 43697-2024):**
- **定级(三级):** 一般数据 / 重要数据 / 核心数据(按遭篡改、泄露、非法利用后的危害程度)。
- **分类(数据域):** 研发数据域、生产数据域(控制信息/工况/工艺参数)、运维数据域、管理数据域(客户与产品/供应链)、外部数据域。

**三档边界规则(级别 → 能否外发):**

| 数据级别 | external_api 外发 | 依据 |
|---|---|---|
| 核心数据 | 绝对禁止 | 法定最高敏感级 |
| 重要数据 | 禁止 | 向境外提供重要数据须申报数据出境安全评估;外部 API 多在境外,产品层直接禁掉最干净 |
| 一般数据 | 可外发(仍须脱敏) | 风险可控,过滤掉可识别标识 |

**默认数据类型表(级别由客户按已备案目录可配置,以下为默认值):**

| 数据类型 | 默认级别 | 外发 |
|---|---|---|
| 合规/认证知识(法规、标准条文) | 一般(本身公开) | ✅ 可发 |
| 合规判断/答案(某客户某零件的认定) | 重要 | ❌ 禁 |
| 工艺参数、控制信息、FA 根因 | 重要/核心 | ❌ 禁 |
| 研发设计数据、图纸 | 重要/核心 | ❌ 禁 |
| ECN/PCN 变更内容 | 重要 | ❌ 禁 |
| BOM、供应链关系 | 重要 | ❌ 禁 |
| 客户标识、零件号、人员信息 | 重要(含个人信息) | ❌ 禁/脱敏 |
| 通用查询模板、非客户特定格式化任务 | 一般 | ✅ 可发(脱敏) |

**关键区分:合规知识(法规文本)可外发;合规判断(针对具体客户/零件的认定)是重要数据,禁止外发。** 核心卖点是后者那个判断,恰是最不能外泄的一类——router 必须分清知识层(可发)与判断层(禁发)。

**三条落地纪律(同时是法规硬要求):**
1. **入库即打标,不在 router 临时判断。** Memory/RAG 入库时给每条数据打 `data_level` + `data_domain` 标签,router 只读标签拦截。
2. **外发留审计日志。** 哪条数据、什么级别、发去哪、谁触发,全记;既是 ops 需要,也是合规证据。
3. **级别让客户拍板,产品只提供机制。** 分类策略客户可配置,默认用上表,由客户数据安全负责人按已备案目录调整。

**重点企业例外:** 若客户在「工业领域数据安全风险防控重点企业名录」内(掌握关键核心技术、关系产业链安全的大型企业,如 Bosch 类大概率在内),默认应将 `escalation_target` 设为 `local_large` / `human`,**整体关闭 external_api**,而非靠分类过滤放行——分类过滤是给中小客户的弹性,不是给高敏感客户开口子的理由。

## 7. 自建 vs 借

| 类别 | 内容 | 说明 |
|---|---|---|
| **自建(资产)** | 工业文档解析硬化 | RAGFlow 之上的工业专项强化 |
| **自建(资产)** | 合规/变更属性图 | 人工策展,跨客户复用,最强护城河 |
| **自建(资产)** | 工业 Skill 库 | ECN/PCN、合规、FA、BOM,复用载体 |
| **自建(适配)** | SKILL.md 加载器 | Pydantic AI 上补 |
| **自建(界面)** | 业务监控台 | Temporal UI 之上,可后做 |
| 借 | 其余全部 | Mem0/RAGFlow/AGE/Qwen3/Temporal/Pydantic AI/MCP |

## 8. 关键约束

- **部署:** 私有化、可断网、数据不出厂。
- **模型:** 主体本地小模型;升级口可选,但工业客户私有化场景默认不通外部 API。
- **硬件基线:** RTX 4090 24GB 级单机起步。
- **License:** 全栈 Apache/MIT,无 copyleft。两个要绕:Neo4j(GPLv3,改用 AGE)、Dify(自定义协议,已砍)。
- **可追溯:** 合规相关答案必须带来源、可审计。

## 9. 成功标准

- **私有化可跑:** 在客户内网、断网条件下跑通一个完整工业工作流(以 ECN/PCN 为首发)。
- **小模型够用:** 本地 Qwen3 在结构化/模板化任务上达到工业可接受准确率,硬 case 走升级口 < 一定比例。
- **可追溯:** 合规相关答案 100% 可回溯到来源文档+版本+章节。
- **复用率:** 第二个客户落地时,核心资产复用 ≥ 60%,差异主要在组合。
- **不猜:** 无来源支撑时正确弃权 / 转人工,而非编造。

## 10. 路线(粗)

1. **P0 — 单层验证:** 各组件本地跑通最小验证(Pydantic AI 驱动 Qwen3 + 挂一个 MCP 工具;RAGFlow 解析一批真实工业文档;AGE 建一张小合规图)。
2. **P1 — 首个场景闭环:** ECN/PCN 变更管理端到端(解析→图→检索→小模型抽取→Temporal 编排→可追溯输出),私有化部署。
3. **P2 — 资产沉淀与复用:** 把 P1 的解析规则、合规图、Skill 抽象成可复用核心,第二个客户验证复用率。
4. **P3 — 三边扩展:** 向选型/报价、市场准入/合规延伸,逼近三边决策图终局。

## 11. 风险与待定

- **AGE 索引调优** 是图层唯一会真出问题的点,P0 就要把索引策略当正事做。
- **SKILL.md 加载器** 要自建,别到设计阶段忘了排期。
- **小模型真实上限**:无预结构化的长链多跳 / 合规判断题 / 长篇合成,小模型做不好;router 升级口的触发边界与成本要在 P1 量化。
- **AGE 母公司**(SKai Worldwide)重心转移,每年回看一次供应链风险。
- **✅ 已定(原 router 待拍):** 升级口为按客户分级的可配置开关 `escalation_target`(`local_large` / `external_api` / `human`),由客户数据外发策略决定。可接受外发的客户用 OpenAI 兼容 API(成本低);硬密封客户只走内网更大本地模型或转人工。切换仅配置变更,不动架构。
- **✅ 已定(原数据边界待拍):** 跨边界规则对齐工信部工业数据三级分类(一般/重要/核心)——核心/重要禁外发,一般可脱敏外发;入库即打标、外发留审计、级别客户可配置;重点企业默认整体关闭 external_api。详见 §6.8。
